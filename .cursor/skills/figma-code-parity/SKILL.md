# Figma ↔ код (React + CSS variables)

## Когда применять

- Пользователь просит собрать макет из компонентов, совпадающих с кодом.
- Нужно понять, как синхронизировать Figma и `src/`, токены, варианты.
- Речь о Code Connect, захвате страницы в Figma, библиотеке компонентов.
- Любая формулировка вроде «макет как в браузере», «как в репозитории», «дизайн-система и код».

**Перед `use_figma`:** обязательно загрузить навык `figma-use` (или эквивалент из Cursor) — правила Plugin API.

---

## Принцип 1:1

| В коде | В Figma |
|--------|---------|
| Один файл компонента, например `src/components/common/Button.jsx` | Один главный компонент **Button**, не дубликаты «на глаз» |
| Пропсы (`variant`, `size`, …) | **Variants** с теми же осями и значениями (например `Style=Primary\|Secondary`, `Size=SM\|MD\|LG`) |
| Стили из `variables.css` / классов | Визуал через **переменные** Figma с WEB code syntax `var(--имя)` как в проекте |

Так контракт в Dev Mode совпадает с тем, что ждёт разработчик.

---

## Токены

- Источник в коде: `src/styles/variables.css` (и импорты).
- В Figma: у семантических переменных задавать **Code syntax (WEB)** в виде `var(--color-primary)`, `var(--spacing-md)` и т.д. — **с обёрткой `var()`**.
- Избегать «сырых» hex в компонентах там, где в коде уже есть токен; новые значения — по возможности добавлять в CSS и дублировать переменной в Figma.

---

## Экраны только из инстансов

- Страницы (дашборд и т.д.) собирать из **инстансов** компонентов библиотеки (Sidebar, TopBar, Table, Pagination …).
- Не воспроизводить те же блоки отдельными прямоугольниками — иначе расхождение с кодом неизбежно.
- **Полный захват страницы** (Figma MCP `generate_figma_design` / html-to-design + скрипт в `index.html`) даёт **референс как в браузере**, но **не заменяет** библиотеку: для «собрано из компонентов» экран нужно пересобрать из инстансов поверх или вместо голых слоёв захвата.

---

## Code Connect (явная связь Figma ↔ файл)

1. Официальный инструмент Figma: [Code Connect](https://www.figma.com/developers/code-connect).
2. В репозитории: `figma.config.json`, рядом с компонентами — аннотации `figma.connect` (путь к модулю, сопоставление вариантов Figma пропам React).
3. Публикация/синхронизация через CLI (`figma connect publish` или актуальная команда из документации).
4. Дополнительно: MCP **`add_code_connect_map`**, если нужно вручную связать `node-id` и путь к компоненту.

---

## Порядок сборки библиотеки

1. Атомы: иконка/плейсхолдер, IconButton, Button, поля ввода.
2. Молекулы: SearchInput, Pagination, ячейки/строки таблицы.
3. Лейауты: Sidebar, TopBar, Header.
4. Страницы: только композиция инстансов.

Изменение API в коде (новый вариант/проп) → то же изменение в Figma (оси/значения variants).

## Инструменты MCP (кратко)

| Задача | Инструмент |
|--------|------------|
| Правки в файле, переменные, компоненты | `use_figma` (после навыка figma-use) |
| Слой «как в браузере» в существующий файл | `generate_figma_design`, `outputMode: existingFile`, `fileKey` из URL файла |
| Поиск в библиотеках | `search_design_system` |
| Связь узла с кодом | `add_code_connect_map` / Code Connect CLI |

## Сводная таблица целей

| Цель | Механизм |
|------|----------|
| Пиксельно как в браузере | Захват html-to-design + при необходимости токены |

| Формальное соответствие файлам в репо | **Code Connect** + те же имена вариантов и токенов |

## Практический чеклист синхронизации (репо ↔ Figma)

1. **Токены:** сверить `src/styles/variables.css` с коллекциями Pyramid в файле; добавить семантические переменные с `var(--…)` в WEB code syntax.
2. **Компоненты:** на странице Library держать мастера под `src/components/**`; экран «Synced — code (instances)» — только инстансы, без «рисования с нуля».
3. **Пиксели как в браузере:** `generate_figma_design` + `existingFile` + `fileKey`, скрипт в `index.html`, открыть `localhost…/pyramid/#figmacapture=…`.
4. **Code Connect:** `add_code_connect_map` / CLI — только при **Developer seat** на плане Organization/Enterprise; иначе связь узел↔файл в Dev Mode недоступна через API.

## Чего не делать

- Не вызывать `use_figma` без чтения навыка с правилами Plugin API.
- Не параллелить несколько `use_figma` подряд для одного файла — только последовательно.
- Не путать «скрин в Figma» с «библиотекой компонентов»: для второго нужны отдельные компоненты и инстансы.
