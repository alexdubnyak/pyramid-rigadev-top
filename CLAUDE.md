# Правила работы с проектом

## Работа с Figma

При любом обращении к Figma (чтение макета, перенос дизайна в код, сверка дизайна):

1. **Всегда используй локальный Figma Dev MCP** для работы с текущим выделением в Figma. Не требуй ссылку на файл или фрейм: пользователь работает с локальным Figma Dev MCP, и выделенный фрейм нужно читать напрямую через него. Локальный сервер работает через `http://127.0.0.1:3845/mcp` (`figma-dev-mode-mcp-server` / `figma-desktop`); для `get_design_context`, `get_metadata`, `get_screenshot` не передавай `fileKey`, а `nodeId` можно не передавать — тогда используется текущий selection.

2. **Всегда используй Figma Inspect** (`get_design_context`, `get_metadata`, `get_screenshot`) для получения точных данных о макете — размеров, цветов, отступов, шрифтов, стилей.

3. **Внедряй ровно то, что есть в макете** — никогда не придумывай элементы, стили, размеры или поведение «от себя». Все значения должны соответствовать тому, что реально нарисовано в Figma.

4. **Если возникают сомнения — спрашивай пользователя.** Не делай предположений о:
   - Какую иконку использовать, если в макете неочевидно
   - Как должен вести себя hover/active/disabled если в макете этого нет
   - Какой asset подставить если он не найден
   - Как адаптировать компонент под существующую систему проекта, если есть конфликт

5. **Ассеты сохраняй в папку проекта** (`src/assets/` или аналогичную для этого проекта). Скачивай изображения и SVG через `download_figma_images`, сохраняй локально и импортируй через обычные относительные пути или `getAssetPath`.

6. **Никогда не используй прямые импорты изображений из Figma** (CDN ссылки вида `figma.com/...`, `figma-alpha-api.s3...`, `s3-alpha-sig.figma.com` и т.п.). Такие ссылки временные и нестабильные — они ломаются через короткое время. Всегда сначала скачивай ассет локально.

## Общие принципы

- Предпочитай редактирование существующих файлов созданию новых.
- Не добавляй фичи, рефакторинг или абстракции сверх того, что требует задача.
- При UI-изменениях проверяй результат в браузере на запущенном dev сервере.
