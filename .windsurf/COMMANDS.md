# Команды для запуска ролей

Этот файл содержит готовые команды для копирования и вставки в чат.

---

## 🔄 Автоматический режим (рекомендуется)

### `/loop` - Полный автоматический цикл

**Просто введите:**
```
/loop
```

**Или скопируйте полную команду:**

```
Я запущу полный цикл внедрения макета из Figma с автоматическим выполнением всех ролей.

Шаг 1: Orchestrator (Анализ)
Я - Orchestrator. Моя задача:
1. Получить ВЕСЬ файл из Figma через mcp0_get_design_context (без nodeId)
2. Определить все компоненты в файле (Sidebar, TopBar, Main Content, DashboardPage, и т.д.)
3. Получить детальный дизайн для КАЖДОГО компонента
4. Проанализировать все компоненты, цвета, размеры, отступы
5. Сравнить с текущей реализацией
6. Создать детальную спецификацию в /.windsurf/figma-spec.md для ВСЕХ компонентов
7. Передать задачу Developer

Начинаю работу с анализа ВСЕГО макета Figma...

После завершения Orchestrator автоматически перейду к Developer.

---

Шаг 2: Developer (Реализация)
Я - Developer. Моя задача:
1. Прочитать спецификацию из /.windsurf/figma-spec.md
2. Реализовать все изменения согласно приоритетам (P0 → P1 → P2)
3. Использовать CSS переменные
4. Сохранить существующие паттерны
5. Создать детальный отчет в /.windsurf/dev-report.md
6. Передать результат QA

Начинаю работу с чтения спецификации...

После завершения Developer автоматически перейду к QA.

---

Шаг 3: QA (Проверка)
Я - QA. Моя задача:
1. Прочитать спецификацию из /.windsurf/figma-spec.md
2. Прочитать отчет Developer из /.windsurf/dev-report.md
3. Получить эталонные значения из Figma
4. Проверить каждый компонент на соответствие
5. Создать детальный отчет в /.windsurf/qa-report.md
6. Передать результат Orchestrator

Начинаю работу с изучения документации...

После завершения QA автоматически перейду к финальному решению Orchestrator.

---

Шаг 4: Orchestrator (Финальное решение)
Я - Orchestrator. Моя задача:
1. Прочитать QA отчет из /.windsurf/qa-report.md
2. Принять решение: завершить или начать новую итерацию
3. Если есть несоответствия - создать новую спецификацию
4. Если все хорошо - завершить workflow

Начинаю работу с анализа QA отчета...

Если статус FAIL - автоматически запущу новую итерацию (максимум 3 итерации).
```

**Что произойдет:**
- ✅ Автоматически выполнятся все 4 шага
- ✅ Создадутся все отчеты
- ✅ Код будет изменен
- ✅ Если FAIL - автоматически запустится новая итерация
- ✅ Вы получите финальный результат

**Время выполнения:** 25-80 минут (в зависимости от количества итераций)

```

```

- /.windsurf/dev-report.md
- /.windsurf/qa-report.md
- /.windsurf/implementation-complete.md

## 🚨 Команды для экстренных случаев

### Workflow завис
```
Проанализируй текущее состояние:
1. Прочитай все файлы в /.windsurf/
2. Определи на каком этапе остановились
3. Предложи как продолжить
```

### Нужно начать заново
```
Выполни полную очистку:
1. Удали все файлы в /.windsurf/ кроме workflows/ и roles/
2. Подтверди очистку
3. Запусти Orchestrator (Итерация 1)
```

### Нужно откатить изменения Developer
```
Я хочу откатить изменения Developer:
1. Покажи список измененных файлов из /.windsurf/dev-report.md
2. Для каждого файла покажи что было изменено
3. Предложи как откатить (git или вручную)
```

## 💡 Советы по использованию команд

1. **Копируйте полностью** - команды содержат важные детали
2. **Ждите завершения** - не прерывайте роль в процессе работы
3. **Читайте отчеты** - они содержат важную информацию
4. **Следуйте порядку** - Orchestrator → Developer → QA → Orchestrator
5. **Проверяйте статус** - используйте диагностические команды
