## ⚡ Команды для копирования (ручной режим)

### 1️⃣ Orchestrator (Анализ)
```
Я - Orchestrator. Моя задача:
1. Получить ВЕСЬ файл из Figma через mcp0_get_design_context (без nodeId)
2. Определить все компоненты в файле (Sidebar, TopBar, Main Content, DashboardPage, и т.д.)
3. Получить детальный дизайн для КАЖДОГО компонента
4. Проанализировать все компоненты, цвета, размеры, отступы
5. Сравнить с текущей реализацией
6. Создать детальную спецификацию в /.windsurf/figma-spec.md для ВСЕХ компонентов
7. Передать задачу Developer

Начинаю работу с анализа ВСЕГО макета Figma...
```

### 2️⃣ Developer (Реализация)
```
Я - Developer. Моя задача:
1. Прочитать спецификацию из /.windsurf/figma-spec.md
2. Реализовать все изменения согласно приоритетам (P0 → P1 → P2)
3. Использовать CSS переменные
4. Сохранить существующие паттерны
5. Создать детальный отчет в /.windsurf/dev-report.md
6. Передать результат QA

Начинаю работу с чтения спецификации...
```

### 3️⃣ QA (Проверка)
```
Я - QA. Моя задача:
1. Прочитать спецификацию из /.windsurf/figma-spec.md
2. Прочитать отчет Developer из /.windsurf/dev-report.md
3. Получить эталонные значения из Figma
4. Проверить каждый компонент на соответствие
5. Создать детальный отчет в /.windsurf/qa-report.md
6. Передать результат Orchestrator

Начинаю работу с изучения документации...
```

### 4️⃣ Orchestrator (Решение)
```
Я - Orchestrator. Моя задача:
1. Прочитать QA отчет из /.windsurf/qa-report.md
2. Принять решение: завершить или начать новую итерацию
3. Если есть несоответствия - создать новую спецификацию
4. Если все хорошо - завершить workflow

Начинаю работу с анализа QA отчета...
```

## 🔄 Процесс

```
Orchestrator → Developer → QA → Orchestrator
     ↓            ↓         ↓         ↓
 spec.md     report.md  qa.md    PASS/FAIL
                                      ↓
                              PASS → Готово
                              FAIL → Итерация 2
```

## 📊 Критерии

| Роль | Критерий | Значение |
|------|----------|----------|
| Orchestrator | Все компоненты | 100% |
| Developer | Реализовано | 100% |
| QA | Цвета | ±1 HEX |
| QA | Размеры | ±2px |
| QA | Соответствие | ≥95% |

## 📁 Файлы

| Файл | Создает | Читает |
|------|---------|--------|
| `figma-spec.md` | Orchestrator | Developer, QA |
| `dev-report.md` | Developer | QA, Orchestrator |
| `qa-report.md` | QA | Orchestrator |
| `implementation-complete.md` | Orchestrator | - |

## 🎯 Приоритеты

- **P0** - Критично (блокирует)
- **P1** - Высокий (важно)
- **P2** - Средний (можно отложить)

```
3. Предложи как продолжить
```
