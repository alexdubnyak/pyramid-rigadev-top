# Loop - Автоматический цикл внедрения

Этот workflow автоматически запускает всю цепь ролей: Orchestrator → Developer → QA → Orchestrator (решение).

## Как использовать

Просто введите в чат:
```
/loop
```

Или скопируйте команду ниже:

---

## 🔄 Полный цикл внедрения макета

Я запущу полный цикл внедрения макета из Figma с автоматическим выполнением всех ролей.

### Шаг 1: Orchestrator (Анализ)

Я - Orchestrator. Моя задача:
1. Получить ВЕСЬ дизайн из Figma через mcp0_get_design_context (без nodeId для корневого фрейма)
2. Проанализировать структуру всего файла и определить все компоненты
3. Получить детальный дизайн для КАЖДОГО компонента (Sidebar, TopBar, Main Content, DashboardPage, и т.д.)
4. Извлечь все цвета, размеры, отступы из ВСЕХ компонентов
5. Сравнить с текущей реализацией
6. Создать детальную спецификацию в /.windsurf/figma-spec.md для ВСЕХ компонентов
7. Передать задачу Developer

Начинаю работу с анализа ВСЕГО макета Figma...

---

*После завершения Orchestrator автоматически перейду к Developer*

### Шаг 2: Developer (Реализация)

Я - Developer. Моя задача:
1. Прочитать спецификацию из /.windsurf/figma-spec.md
2. Реализовать все изменения согласно приоритетам (P0 → P1 → P2)
3. Использовать CSS переменные
4. Сохранить существующие паттерны
5. Создать детальный отчет в /.windsurf/dev-report.md
6. Передать результат QA

Начинаю работу с чтения спецификации...

---

*После завершения Developer автоматически перейду к QA*

### Шаг 3: QA (Проверка)

Я - QA. Моя задача:
1. Прочитать спецификацию из /.windsurf/figma-spec.md
2. Прочитать отчет Developer из /.windsurf/dev-report.md
3. Получить эталонные значения из Figma
4. Проверить каждый компонент на соответствие
5. Создать детальный отчет в /.windsurf/qa-report.md
6. Передать результат Orchestrator

Начинаю работу с изучения документации...

---

*После завершения QA автоматически перейду к финальному решению*

### Шаг 4: Orchestrator (Финальное решение)

Я - Orchestrator. Моя задача:
1. Прочитать QA отчет из /.windsurf/qa-report.md
2. Принять решение: завершить или начать новую итерацию
3. Если есть несоответствия - создать новую спецификацию
4. Если все хорошо - завершить workflow

Начинаю работу с анализа QA отчета...

---

## 📊 Результаты цикла

После завершения всех шагов вы получите:

✅ **Файлы**:
- `/.windsurf/figma-spec.md` - Спецификация изменений
- `/.windsurf/dev-report.md` - Отчет разработчика
- `/.windsurf/qa-report.md` - Отчет тестировщика
- `/.windsurf/implementation-complete.md` - Финальный отчет (если PASS)

✅ **Измененный код**:
- Обновленные CSS файлы
- Обновленные компоненты
- Новые CSS переменные

✅ **Статус**:
- PASS → Макет внедрен успешно
- FAIL → Требуется новая итерация (автоматически запустится)

---

## 🔁 Автоматические итерации

Если QA вернет статус FAIL, я автоматически:
1. Увеличу номер итерации
2. Создам новую спецификацию с фокусом на несоответствия
3. Запущу Developer для исправлений
4. Запущу QA для повторной проверки
5. Приму финальное решение

Цикл будет повторяться до получения статуса PASS или максимум 3 итерации.

---

## ⚙️ Настройки

По умолчанию:
- Максимум итераций: 3
- Допуск цветов: ±1 HEX
- Допуск размеров: ±2px
- Минимальное соответствие: 95%

---

## 🎯 Ожидаемое время

- 1 итерация: 25-50 минут
- 2 итерации: 40-80 минут
- 3 итерации: 60-120 минут
